Hyper Text Mark-up Language

What is HTML?

The Hypertext Markup Language(HTML) is one of the two core technologies used for web developing, developed by Sir Tim Berners-Lee in late 1990 after having wrote the browser and server software. From HTML 1.0, the Hypertext Markup Language has had many updates and improved versions throughout the past decade.

HTML 1.0 - The very first release of the Hypertext Markup Language.

HTML 2.0 - HTML 2.0 was the expansion of the Hypertext Markup Language. In relation to the first HTML version released, and was created for the means of being a Web Standard. It was necessary to have a strict set of standards that outlined what the HTML was capable of doing for the developer. This information became available in the development of HTML 2.0, and at the time of its release, it was the source accessible for users to learn exactly how the language worked. HTML 2.0 published as IETF RFC 1866.

HTML 3.0 - intended to be a proposal for the extension of the Hypertext Markup Language, however, the difference of HTML 2.0 and HTML 3.0 was apparently so large that the deployment and standardization of the language became too difficult to manage. Thus, it is no longer being maintained.

HTML 3.2 - (W3C Recommendation, Jan 1997) Relative to HTML 2.0, this HTML version deploys features such as the tables, applets, image text flow, superscript, and subscript.

HTML 4.0 - (W3C Recommendation, Dec 1999) According to the World Wide Web Consortium, HTML 4 is a Standard Generalized Markup Language application that is widely regarded as the standard publishing language of the World Wide Web. It has remained stable since the 1980s in regards to the fact that this markup language is both feature-rich and is flexible.
Differences with HTML 4

  1. Documents must be well-formed. This essentially means that all elements must have closing tags, or at least be enclosed in a special tag.

  2. Element and attribute names must be in lower case.

  3. End tags are required for non-void elements.

  4. Attribute values must always be quoted, even for numerical elements.

  5. Attribute-value pairs must be written in full.

  6. Void elements should also have an end tag.

  7. One or more white-space characters must be mapped to a single inter-word space

A lot of HTML data and content may require you to enter the repeated data time and time again, and this may prove to be time-consuming and inefficient. As such, preprocessors exist to ease the developing of HTML documents, where many contain redundant information that in turn will have to be managed.
