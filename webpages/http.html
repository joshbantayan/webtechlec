Hypertext Transfer Protocol

What is HTTP?

The Hypertext Transfer Protocol(HTTP) is the network protocol standard used by the World Wide Web that defines commands and services used for transmitting web page data. It was invented by Tim Berners-Lee at CERV in 1989. It is an application protocol for distributed, collaborative, and hypermedia information systems that uses a server-client model.

HTTP Fundamentals

-Runs on top of TCP/IP

-Uses TCP port 80 by default

-Uses TCP port 443 (for HTTPS)

HTTP Extensions

Can be extended by defining new requests methods, message headers, and status codes to implement servers and/or clients who will utilize the new functionality

HTTP Resource Addressing

There is a need for a universal syntax in regards to obtaining information or resources in protocols and systems. For HTTP, resources are identified via the URIs.

Standard HTTP Request Methods

GET - Requests using GET retrieve data, and also requests a representation of the specified source.
POST - The POST method is used to send requests to a web server to accept data in the body of the request message, which is intended for storage purposes.
HEAD - This method requests for an identical response to that of a GET request, only excluding the response body.
OPTIONS - This OPTIONS method represents a request for information regarding the communication options available for the trusted source or managed resource.
PUT - This method takes the place of all current representations of the managed resource; it requests that the enclosed entity be stored under the supplied request-URI. The enclosed entity should be considered a modified version of the original, only if under the case that the request-URI happens to refer to an existing resource.
DELETE - The DELETE method simply deletes the resource specified.
CONNECT - This method establishes a two-way communications with the requested resource, and opens up a tunnel to the server identified by the target resource.
TRACE - The TRACE method is used as a debugger to perform a message loop-back test to the user.

HTTP Method Properties

Safe Methods - This is the property wherein the client does not request or expect any state change on the origin server output.
Idempotent Methods - This refers to the intended effect on the server where it would be identical when a single request is issued.
Under the idempotent method falls the following request methods: GET, HEAD, OPTIONS, TRACE, PUT, DELETE Cacheable Methods - This property indicates that the response to a method is allowed to be stored for future reuses Under the cacheable method falls the following request methods: GET, HEAD, POST
