JavaScript Type Conversions and Functions

Implicit Type Conversions

If a statement that JavaScript is operating on has the wrong data types, it implicitly converts the the resulting value to a type that JavaScript thinks is right.

Examples:
nikki paki lagay sa webmaker
19 + null // this returns 19
“20” - 10 // this returns 10 converting “20” to 20
“20” * “10” // this returns 200 converting “20” and “10” to 20 and 10
“Sig” + null // this returns “signull” converting null to “null”

Equality and Identity

The equality operator will attempt to make the data types the same before making the comparison. On the other hand, the identity operator requires both data types to be the same as a prerequisite.

Examples:

paki lagay sa webmaker

1 == true //true
True == true // true
1 === true //false
True === true //true

Functions are blocks of code that are designed to perform a specific task. A function performs its task once called upon. They can be nested and be passed as arguments for other functions.
